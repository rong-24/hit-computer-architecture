## 一、项目简介

计算机体系结构是计算机专业的一门核心课，但在很多学校的培养方案中并不是「统一标配」，考研 408 对本课内容的覆盖也比较有限，因此**系统、成体系的复习资料相对较少**。

本仓库主要记录我在哈工大修读《计算机体系结构》课程期间的：

- 复习思路与时间安排
- 习题整理与考试题复现
- 一些个人理解与易错点总结

希望能为之后的同学提供一个参考框架，而不仅仅是一堆零散的题目。

## 二、主要参考资料

**推荐教材 / 习题册：**

- 张晨曦《计算机系统结构学习指导与题解》  
  - 内容整体**严谨、精炼**，非常适合作为配套习题册  
  - 但书中仍存在个别疏漏与细节问题，使用时建议：
    - 遇到存疑之处优先以本校课堂讲义 / 任课老师的说法为准
    - 重要结论尽量用多道题交叉验证一下

**使用方式建议：**

- 以**做课后题为主线**，通过题目来检查与巩固知识点
- 对于某些「算数题」背后其实考察概念的，要刻意反推：
  - 这道题究竟在检查哪个概念？
  - 如果数算错了，但思路正确，在考试中会扣多少分？

## 三、我的复习路径（仅供参考）

> 个人复习起始时间：**考前一周**

### 1. 第 1–2 天：过一遍 PPT + 网课

- 哈工大课件内容与 B 站上**国防科大计算机体系结构课程**高度重合  
- 用约两天时间：
  - 对照学校 PPT 看完一遍对应网课
  - 目标是**构建整体框架**，而不是抠细节

### 2. 第 3 天：扫描习题 + 集中答疑

- 用一天时间「**走马观花**」式地过一遍张晨曦书中的习题：
  - 一边看题、一边看答案，大致感受考察方式
  - 把看不懂 / 有疑问的题目记录下来
- 将有争议或难点的题目整理后，找老师进行了一次**集中答疑**

### 3. 第 4–5 天：遮答案重做一遍

- 用两天时间，**在不看答案的情况下**重新做一遍习题：
  - 这一步主要是检测：哪些内容真的掌握了，哪些只是「看答案时觉得自己懂了」
  - 做题时尽量模拟考试环境，控制时间、控制草稿量
- 这轮结束后，我对重点概念和典型题型的把握有了比较明显的提升

## 四、复习提醒与命题风格

如果是**刘宏伟老师命题**（以 2025 年为例），请特别注意：

- 整体**偏重概念理解**而非纯计算：
  - 很多题目本质是在问「是否真正理解某个概念」
  - 单纯记公式、会算数但概念模糊，在这一类卷子上会吃亏
- 在 2025 年的试卷中：
  - **大量题目直接或隐含地考察概念定义、条件、适用场景**
  - 做题时要刻意问自己一句：「我真的知道这几个名词的区别吗？」

因此，复习策略上建议：

- **先搞清楚“在讲什么”**，再去练习“怎么算”
- 尤其是流水线、Cache、指令级并行、分支预测等章节：
  - 不要只停留在套公式，而要能**用自己的话解释现象和设计动机**
